# Product

## Register

product

## Users

Axis is for developers who prefer to work with a coding agent from the terminal. They use it in long, focused sessions where the interface should stay out of the way while keeping agent state, tool activity, and approvals legible.

## Product Purpose

Axis provides a personal, local-first coding-agent workflow through print and interactive TUI modes. The interface succeeds when developers can read the conversation, understand agent activity, and act on prompts without leaving their terminal context.

## Brand Personality

Restrained, capable, and terminal-native. Axis should feel calm and precise rather than decorative, with the confidence of a mature developer tool.

## Anti-references

Avoid opaque application shells that fight the user's terminal theme, decorative glass effects, low-contrast muted text, saturated chrome, and visual treatments that make the TUI feel like a web dashboard forced into a terminal.

## Design Principles

- Respect the terminal as the host environment.
- Keep the conversation and current action visually dominant.
- Use emphasis only for focus, state, and risk.
- Preserve clarity during long, information-dense sessions.
- Make configuration reversible and explicit.

## Accessibility & Inclusion

Maintain high legibility for body text, controls, focus, selection, errors, and approval states. Do not rely on color alone for meaning. Preserve keyboard-first operation and reduced-motion behavior where motion is present.
